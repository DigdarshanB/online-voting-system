# Work Breakdown Structure (WBS) — Online Voting System

---

## 1. Project Management

- **1.1** Project Initiation
- **1.2** Sprint Planning & Backlog Management
- **1.3** Progress Monitoring & Reporting
- **1.4** Final Documentation & Submission

---

## 2. Research & Domain Analysis *(Sprint 1)*

- **2.1** Problem Context Analysis
  - 2.1.1 Study traditional voting system issues in Nepal
  - 2.1.2 Identify stakeholders and their pain points
- **2.2** Reference System Analysis
  - 2.2.1 Review existing e-voting systems
  - 2.2.2 Benchmark features and security practices
- **2.3** Project Scoping
  - 2.3.1 Define project boundaries and objectives
  - 2.3.2 Select technology stack (FastAPI, React, PostgreSQL, etc.)

---

## 3. System Design & Requirements Engineering *(Sprint 2)*

- **3.1** Requirements Specification
  - 3.1.1 Define functional requirements
  - 3.1.2 Define non-functional requirements (security, scalability, usability)
- **3.2** Agile Artefacts
  - 3.2.1 Write user stories
  - 3.2.2 Create and prioritise product backlog
- **3.3** System Design
  - 3.3.1 Design system architecture (frontend/backend/DB layers)
  - 3.3.2 Create workflow/process diagrams
  - 3.3.3 Design Entity-Relationship Diagram (ERD)

---

## 4. Authentication & Registration Module *(Sprint 3)*

- **4.1** Voter Registration
  - 4.1.1 Registration form and input validation
  - 4.1.2 Document upload functionality
  - 4.1.3 Email/OTP verification flow
- **4.2** Admin Approval Workflow
  - 4.2.1 Admin review and approve/reject voter registrations
  - 4.2.2 Admin portal invitation-based access
- **4.3** User Interface — Initial Build
  - 4.3.1 Voter portal base UI
  - 4.3.2 Admin portal base UI

---

## 5. Election Management Module *(Sprint 4)*

- **5.1** Election Administration
  - 5.1.1 Create and configure elections
  - 5.1.2 Define election types (federal, provincial, local levels)
  - 5.1.3 Schedule and manage election lifecycle
- **5.2** Candidate Management
  - 5.2.1 Add/edit/remove candidates
  - 5.2.2 Associate candidates with elections and constituencies
- **5.3** Voter Management
  - 5.3.1 Assign voters to elections/constituencies
  - 5.3.2 Voter eligibility checks
- **5.4** Voting Process
  - 5.4.1 Ballot casting flow
  - 5.4.2 Multi-level government election execution

---

## 6. Monitoring, Audit & Reporting Module *(Sprint 5)*

- **6.1** Results Management
  - 6.1.1 Real-time vote tallying
  - 6.1.2 Results publication and display
- **6.2** Audit & Transparency
  - 6.2.1 Audit log integration
  - 6.2.2 Tamper-evident audit trail
- **6.3** Reporting
  - 6.3.1 Election summary reports
  - 6.3.2 Export/download report functionality
- **6.4** Admin Dashboard
  - 6.4.1 Election oversight dashboard
  - 6.4.2 Live monitoring views

---

## 7. Security, Testing & Finalisation *(Sprint 6)*

- **7.1** Biometric Security
  - 7.1.1 Liveness detection integration
  - 7.1.2 Face comparison verification before voting
- **7.2** Account Security
  - 7.2.1 Password recovery improvements
  - 7.2.2 Session and token hardening
- **7.3** Testing
  - 7.3.1 Unit testing (backend)
  - 7.3.2 Integration testing
  - 7.3.3 User acceptance testing (UAT)
  - 7.3.4 Security/penetration testing
- **7.4** UI/UX Polish
  - 7.4.1 Usability improvements on voter portal
  - 7.4.2 Usability improvements on admin portal
  - 7.4.3 Responsive design and accessibility fixes
- **7.5** System Finalisation
  - 7.5.1 Bug fixes and last-minute refinements
  - 7.5.2 Deployment preparation
  - 7.5.3 Final review and sign-off
